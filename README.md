Разработка голосового персонального ассистента «Лера» для одноплатного компьютера NVIDIA Jetson с офлайн-распознаванием и синтезом русской речи
---
| Обозначение | Расшифровка |
|-------------|-------------|
| **ASR** | Automatic Speech Recognition — автоматическое распознавание речи |
| **TTS** | Text-to-Speech — синтез речи по тексту |
| **API** | Application Programming Interface — программный интерфейс приложения |
| **RSS** | Rich Site Summary — формат ленты новостей |
| **GPIO** | General-purpose input/output — программируемые линии ввода-вывода |
| **Офлайн-модель** | Модель, работающая без обращения к облачному сервису распознавания |
---
- распознаёт команды **офлайн** (без отправки аудио в облако для ASR);
- отвечает **синтезированной** русской речью;
- выполняет прикладные сценарии: время, погода, краткая сводка новостей, таймеры и будильники;
- при наличии платформы **NVIDIA Jetson** может использовать **GPIO** для индикации режимов (прослушивание, ответ, ошибка) и заложено управление реле/освещением.
- Имя ассистента — **«Лера»**; активация по ключевому слову в распознанной фразе.
---
### Аппаратная составляющая

Система ориентирована на запуск на **NVIDIA Jetson** (или совместимой среде с Python и аудиоустройством). Предусмотрено:

- **микрофон** и **аудиовыход** (через драйвер, с которым работает библиотека `sounddevice`);
- опционально — **GPIO** платы в режиме BOARD: RGB-светодиоды на пинах 11, 13, 15 (индикация режимов) и линия реле/света на пине 16 (заложена в коде; при отсутствии `Jetson.GPIO` программа продолжает работу на ПК без GPIO).
---
### Программная составляющая

- Язык реализации: **Python 3**.
- **Распознавание речи:** библиотека **Vosk**, модель из каталога `model` (компактная русская модель для встраиваемых устройств; в `model/README` приведены ориентиры по качеству на тестовых наборах).
- **Синтез речи:** **Piper** (нейросетевой TTS), голосовая модель `synthModel/mari.onnx` и конфигурация `synthModel/mari.onnx.json` (язык ru-RU, частота дискретизации синтеза 22050 Гц согласно конфигу).
- **Захват аудио:** `sounddevice` с частотой **16000 Гц**, моно, блоками по 8000 отсчётов — согласовано с требованиями Vosk.
- **Логика ассистента:** класс `LeraBrain` — разбор намерений по ключевым словам, таймеры (`threading.Timer`), будильники (список времени и фоновый поток с шагом 30 с), обращения к **OpenWeatherMap** и **RSS Ленты.ру**.
- **Числа в речи:** преобразование цифр и времени в слова — `num2words` (русский язык).
---
### Библиотеки и модели речи

- **Vosk** — движок на базе Kaldi, распознавание на устройстве, низкая задержка при подходящей модели.
- **Piper** — быстрый нейросетевой TTS; модель загружается один раз при старте (`PiperVoice.load`), что снижает задержку при каждом ответе.
- **sounddevice / NumPy** — низкоуровневый ввод-вывод и работа с PCM для воспроизведения WAV после синтеза (заголовок WAV 44 байта отбрасывается перед `sd.play`).
---
### Внешние сервисы и данные

- **OpenWeatherMap** — HTTP API для текущей погоды по выбранному городу (в коде задан Новосибирск); для работы требуется ключ API (хранится в конфигурации разработчика, в пояснительной записке не дублируется).
- **RSS** `https://lenta.ru/rss/news` — разбор ленты через **feedparser**, озвучивание заголовков первых трёх материалов.
---
### Обработка голосовых команд

Класс `LeraBrain` выполняет эвристический разбор текста (нижний регистр, вхождение подстрок):

- **Погода** — ключевое слово «погода»;
- **Новости** — «новости»;
- **Время** — «время», «час», «времени»;
- **Таймер** — «таймер» + извлечение числа словами (`words_to_numbers`) или отмена по ключевым словам;
- **Будильник** — «будильник» + числа для часов/минут, отмена будильников отдельной веткой;
- **Свет / лампа / розетка** — ответы-заглушки «включён/выключен» по ключевым словам (расширяемо до реального GPIO).

При нераспознанной команде возвращается фраза «Не поняла команду.»
